Chat Feature

Our chat feature allows you to engage with different models and manage your conversations efficiently. Here's what you can do:

  • Select a Model: You can choose the model you'd like to interact with from the available options. Each model may have different specialties, so feel free to select the one that suits your needs best.
  • Save a Chat: If you want to revisit a conversation later, you can save the chat session. Keep in mind, once a chat is saved, you will no longer be able to message the model in that specific conversation. It's great for preserving important discussions.
  • Delete a Chat: If you no longer need a saved chat, you can delete it from your records. This will permanently remove the conversation, so make sure it's something you won't need again before deleting.

Make the most of the chat feature by selecting the right model, saving meaningful conversations, and managing your chat history with ease.

Currently, all of our models—both small and large—are available for free and are still under development. You are welcome to try them at no cost during this period of modification and improvement.

Security Indicators

Our Security Indicators feature allows you to create visualizations of vulnerabilities discovered by the pentester. You can either manually enter data or upload an Excel file with the vulnerability information.

Manual Data Entry

Our system allows for manual entry of vulnerabilities through the website forms, with a limit of up to 6 vulnerabilities. This feature is designed to let users input detailed information about the vulnerabilities they have identified. For each vulnerability, users can specify:

  • Vulnerability Name: The name of the vulnerability.
  • Occurrence Count: The number of times this vulnerability was found.
  • Severity: The level of severity of the vulnerability (e.g., High, Medium, Low).

These details will be used to generate visualizations in the pentesting report. Please ensure that only up to 6 vulnerabilities are entered manually; any additional vulnerabilities must be uploaded via an Excel file or other supported methods.

Excel File Upload

To upload an Excel file with vulnerability information, please ensure that the file follows the format outlined below. This will help our system correctly interpret and process the data for visualization. The Excel file should include the following columns:

  • Vulnerability Type: The type of vulnerability (e.g., SQL Injection, XSS).
  • Count: The number of times this vulnerability was detected.
  • Severity: The severity level of the vulnerability (e.g., High, Medium, Low).

Ensure that each column is properly labeled and that the data is accurately filled in according to the specified format. This will ensure that the system can effectively process and visualize the vulnerability data.
